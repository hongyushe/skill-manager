项目概述

Skill Manager 是一个 Next.js 全栈应用，用于管理 ~/.claude/skills/ 目录下的 Claude Code Skills。提供 Web 界面来可视化 Skill 结构、管理定时任务、手动触发执行，并通过 OpenAI 兼容 API 驱动 LLM 完成自动化工作流。

Skill 可视化

自动解析 SKILL.md，哈希追踪自动同步摘要

多种触发方式

Cron、Webhook、手动、Chain 链式

自然语言建任务

中文描述 → LLM 自动解析为结构化配置

MCP 工具集成

通用 MCP 客户端，连接任意 MCP 服务

执行追踪

完整历史、Token 用量、可折叠过程展示

Lobster 通知

结果自动推送到微信、钉钉、Slack 等

  • 保存时检测 ... 占位符，自动保留原值不被覆盖

仪表盘

图：Dashboard — Active Tasks 和 Skill 列表
  • Active Tasks — 汇总所有 Skill 中状态为 active 的任务，显示触发类型图标（蓝色=Cron 绿色=Webhook 白色=Manual）、任务名称、所属 Skill
  • Skills — 有任务的 Skill 卡片网格，显示名称、任务数量和摘要
  • 未配置任务 — 列出没有配置任务的 Skill

自动同步

Dashboard 加载时自动检测所有 SKILL.md 的 SHA-256 哈希。如果 SKILL.md 内容发生变化，或发现新 Skill，自动重新生成摘要。基于哈希对比，避免不必要的 LLM 调用。

Skill 详情页

图：news-monitor Skill 详情 — 摘要、任务、执行结果、历史

摘要（Summary）

三种状态：

  • 无摘要 → 显示"Generate Summary"按钮
  • 有摘要 → 只读显示 + "Edit"按钮
  • 编辑中 → 文本框 + Save/Cancel 按钮

摘要由 LLM 基于 SKILL.md 内容生成，200-500 字纯文本描述。

任务（Tasks）

每个任务卡片包含：触发类型图标、名称、调度表达式、指令预览、▶ Run 和删除按钮。

最新结果（Latest Result）

  • 元信息：时间戳、状态、任务名称、耗时
  • 可折叠过程行：▸ 过程 (N轮推理 · M次工具调用)，点击展开显示每次工具调用详情
  • 最终输出结果

过程数据仅在执行后临时显示，刷新页面后消失，不持久化到 history.json。

执行历史（History Timeline）

  • 默认显示最近 10 条，可展开查看全部
  • 每条记录：时间、任务名/触发来源、状态（颜色区分）、输出摘要
  • 完整的编辑日志存储在 history.json（每次 custom.md 修改的 before/after）

自然语言创建任务

Skill 详情页点击 "+ Add Task" 打开创建向导：

第一步：输入描述

用中文（或英文）描述你想要的任务，例如：

每天早上9点搜索特朗普最新言论，用中文汇总

点击 "Analyze"，LLM 自动解析为结构化配置。

第二步：审核并保存

LLM 返回解析结果，可手动修改：任务名称、触发类型、触发配置、执行指令。点击 "Confirm & Save" 保存。任务 ID 自动递增（TASK-001, TASK-002, ...）。

此功能需要在 Settings 中配置 LLM API Key。

触发方式

类型说明配置方式
cron定时执行，标准 Cron 表达式trigger_config: "0 9 * * *"
webhookHTTP POST 回调，外部系统触发配置 endpoint，如 /api/hooks/my-hook
manualWeb 界面手动点击默认方式，无需配置
chain链式触发，执行完自动触发下一个 Skill配置 chainTarget

Cron 触发（自动注册）

添加/更新 Cron 类型的 Task 时，系统自动注册到系统 crontab，无需手动配置。

0 9 * * * curl -s -X POST http://localhost:3000/api/execute/news-monitor?trigger=cron&task_id=TASK-001 # skill-manager:news-monitor:TASK-001
  • 使用标记注释 # skill-manager:<name>:<taskId> 管理条目
  • 删除 Task 时自动清理对应的 crontab 条目

Webhook 触发

curl -X POST http://localhost:3000/api/hooks/my-hook \
  -H "Content-Type: application/json" \
  -d '{"event": "trigger", "data": "额外上下文"}'

系统扫描所有 Skill 找到匹配的 webhook trigger，将 payload 附加到 Prompt 中执行。

Chain 链式触发

当前 Skill 执行完成后自动触发下一个 Skill。

## Triggers
### chain-1
- type: chain
- chainTarget: next-skill-name
- status: active
  • 最大 5 层链式嵌套深度（防止无限循环）
  • 异步 fire-and-forget，不阻塞当前 Skill
  • 链中某个 Skill 失败不影响其他 Skill

工具系统与 MCP

通用 MCP 客户端

内置 @modelcontextprotocol/sdk，连接任意 Streamable HTTP MCP 服务器：

import { callMcpTool } from "@/lib/mcp-client";

const result = await callMcpTool(
  "https://api.z.ai/api/mcp/web_search_prime/mcp",
  "your-api-key",
  "web_search_prime",
  { search_query: "搜索内容" }
);

每次调用创建新连接（避免会话过期），自动处理初始化握手和 session 清理。

在 Skill 中声明工具

## Tools
- web_search
工具名说明参数依赖
web_search通过 Z.AI MCP 搜索 Webquery: 搜索关键词zai_api_key

工具在执行时由 LLM Agent 自动判断何时调用，无需手动触发。

自定义 Skill 配置

配置存储在 ~/.claude/skills/skill-manager/<name>/custom.md。

新格式（推荐）

## Tools
- web_search

## Tasks
### TASK-001
- name: 任务名称
- trigger_type: cron
- trigger_config: 0 9 * * *
- instruction: |
    具体的任务指令...
- status: active

旧格式（向后兼容）

系统同时支持旧格式，可在同一个 custom.md 中共存：

## Triggers
### TRG-001
- intent: 触发意图
- type: cron
- cron: 0 9 * * *
- status: active

## Execution Details
### EXEC-001
- intent: 执行意图
- prompt: |
    附加执行指令...

数据目录结构

~/.claude/skills/
  news-monitor/
    SKILL.md                              # Skill 定义（Claude Code 读取此文件）
  skill-manager/
    news-monitor/
      custom.md                           # 自定义配置（任务、触发器、工具）
      summary.md                          # LLM 自动生成的摘要
      history.json                        # 执行历史 + 编辑日志
      .skill-hash                         # SKILL.md 的 SHA-256 哈希（变更检测）

skill-manager/ 子目录确保数据文件不干扰 Claude Code 对 SKILL.md 的读取。

API 参考

Skill 操作

方法路径说明
GET/api/skills列出所有 Skill 名称
GET/api/skills/[name]获取 Skill 完整详情
PUT/api/skills/[name]更新 Skill（添加/更新任务、摘要）
DELETE/api/skills/[name]删除任务或触发器

执行

方法路径说明
POST/api/execute/[name]?trigger=manual&task_id=TASK-001执行指定任务
POST/api/execute/[name]?trigger=manual执行整个 Skill
POST/api/hooks/[hookId]Webhook 触发

其他

方法路径说明
GET/api/settings获取配置（API Key 脱敏）
PUT/api/settings更新配置
GET/api/summary批量同步所有 Skill 摘要
POST/api/summary生成单个 Skill 摘要
POST/api/interpret自然语言解析为任务配置
POST/api/cron管理 Crontab（register/remove/list）

技术架构

  • 前端 Next.js App Router + React + Tailwind CSS
  • 执行引擎 @mariozechner/pi-coding-agent（多轮 Agent Loop + Tool Calling）
  • MCP @modelcontextprotocol/sdk（通用 MCP 客户端）
  • 调度 系统 crontab（标记注释管理）

执行流程

用户/定时器/Webhook → API Route (/api/execute/[name])
  → 读取 Skill 配置 (custom.md)
  → 解析工具声明 (## Tools) → resolveTools() → ToolDefinition[]
  → assemblePrompt() 组装完整 Prompt
  → pi-coding-agent Agent Loop
      → LLM 决策 → 调用工具 → 获取结果 → 继续推理（多轮自动）
  → 记录执行日志 (history.json)
  → Lobster 通知（如已启用）
  → 检查 Chain 触发 → 递归执行下一个 Skill
  → 返回结果 + 过程数据 (process steps, reasoning turns)

执行引擎细节

参数值
Provider 类型openai-completions（OpenAI 兼容 API）
上下文窗口128K tokens
最大输出4096 tokens
Token 估算基于字符数（ceil(text.length / 4)）
系统提示词只输出最终结果，不包含推理过程
